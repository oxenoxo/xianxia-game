## 一、Bug 修复记录

### 1.1 血量越玩越低（已修复）

**根因**：`winBattle` 战斗胜利后没有恢复 HP，玩家只能带着残血进入下一场战斗。

**修复**：战斗胜利后恢复 30% 最大生命值。

```javascript
// winBattle 中新增
var healAmt = Math.floor(gameState.maxHp * 0.3);
gameState.hp = Math.min(gameState.maxHp, gameState.hp + healAmt);
```

### 3.2 详细分析

#### 问题 1：战斗收益过低（P0）

**现象**：假设玩家 10 级，打一只同等级灰狼：
- 灰狼（10级缩放）：HP ≈ 50 * (1 + 10*0.08) = 90，攻击 ≈ 8 * (1 + 10*0.05) = 12
- 玩家：HP = 100 + 90 = 190，攻击 = 10 + 18 = 28
- 单次伤害：28 * 1.2 / (1 + 2*0.08) ≈ 29
- 需要 4 次攻击打死灰狼，耗时 3.2 秒
- 获得：经验 ≈ 20 * 1.4 = 28，金币 ≈ 10 * 1.3 = 13
- 同期修炼一次：经验 = 10 * 倍率（约 1-2）= 10-20，耗时 1 次点击

**结论**：战斗耗时 3-5 秒，收益只比修炼高 30-50%，但风险是掉血甚至死亡。**收益风险比不合理**。

**对标**：
- 《寻道大千》：战斗是核心资源获取方式，掉落装备驱动战斗
- 《一念逍遥》：战斗掉落炼丹/炼器材料，是养成必需

**建议**：
1. 战斗经验提升 50-100%
2. 增加战斗专属掉落（装备必掉、材料）
3. 增加"连斩"机制：连续战斗有额外加成

#### 问题 3：敌人强度曲线过陡（P1）

**现象**：敌人 HP 缩放系数 `1 + lv * 0.08`，攻击 `1 + lv * 0.05`。

| 等级 | 灰狼 HP | 灰狼攻击 | 玩家 HP | 玩家攻击 |
|------|---------|---------|---------|---------|
| 1 | 54 | 8 | 100 | 10 |
| 10 | 90 | 12 | 190 | 28 |
| 20 | 130 | 16 | 290 | 48 |
| 50 | 250 | 28 | 590 | 108 |

**问题**：50 级时灰狼 HP 250，玩家攻击 108，需要 3 次攻击。但同期幼龙（Boss）HP = 500 * (1 + 50*0.05) = 1750，攻击 = 50 * (1 + 50*0.03) = 125。

**结论**：高等级时普通怪和 Boss 差距过大，Boss 战变成纯消耗战。

**建议**：
1. 普通怪 HP 缩放降至 `1 + lv * 0.05`
2. Boss HP 缩放提升至 `1 + lv * 0.08`
3. 增加敌人等级上限，避免低等级怪在高等级时过强

#### 问题 4：金币经济失衡（P1）

**现象**：
- 前期：装备价格 200-600，战斗掉落 10-20 金币，需要打 10-30 场才能买一件装备
- 后期：装备价格 15000，但战斗掉落按 `gold * (1 + lv * 0.03)` 增长，50 级时幼龙掉落 = 200 * 2.5 = 500，仍需要 30 场

**问题**：金币获取线性增长，装备价格指数增长，玩家永远买不起高级装备。

**建议**：
1. 金币掉落公式改为 `gold * (1 + lv * 0.08)`
2. 增加金币副本/活动
3. 装备出售价格提升，让玩家可以通过倒卖低级装备积累资金

#### 问题 5：技能性价比不均（P1）

| 技能 | 效果 | 成本 | 性价比评估 |
|------|------|------|-----------|
| 火球术 | 1.8-3.8 倍攻击 | 低 | 高，战斗核心 |
| 治愈术 | 30-165 恢复 | 低 | 中，后期鸡肋 |
| 冥想 | 1.1-2.0 倍修炼 | 中 | **极高**，永久被动 |
| 力量强化 | +3-30 攻击 | 高 | 低，不如直接加属性点 |
| 护盾术 | +2-20 防御 | 中 | 低，防御收益递减 |
| 双倍经验 | 1.5-3.5 倍经验 | 高 | **极高**，但和冥想叠加过强 |

**问题**：冥想和双倍经验是乘法叠加，同时点满可达 7 倍经验，成长曲线崩坏。

**建议**：
1. 冥想改为加法：`基础 * (1 + 0.1 * level)` → `基础 + 10% * level`
2. 双倍经验和冥想互斥或改为取最大值
3. 力量强化和护盾术效果提升 2 倍

## 六、关键数值公式建议

### 6.1 经验曲线

当前：`expToNext *= 1.3`

建议：改为分段增长，前期快后期慢：
```javascript
// 1-20级：1.2倍
// 21-50级：1.3倍
// 51+级：1.4倍
var rate = gameState.level <= 20 ? 1.2 : gameState.level <= 50 ? 1.3 : 1.4;
gameState.expToNext = Math.floor(gameState.expToNext * rate);
```

### 6.2 伤害公式

当前：`atk * 1.2 / (1 + def * 0.08)`

建议：增加保底伤害和防御收益上限：
```javascript
var reduction = def * 0.08 / (1 + def * 0.08); // 渐近 100%，但永远达不到
damage = Math.max(1, Math.floor(atk * 1.2 * (1 - reduction) + random(-2, 3)));
```

### 6.3 装备掉落

```javascript
Game.prototype.rollEquipDrop = function() {
    var luckBonus = gameState.luck * 0.01;
    var baseRate = 0.4 + luckBonus; // 基础 40%
    if (Math.random() > baseRate) return null;
    
    var qualityRoll = Math.random();
    var quality;
    if (qualityRoll < 0.5 + luckBonus) quality = 'common';
    else if (qualityRoll < 0.8 + luckBonus) quality = 'uncommon';
    else if (qualityRoll < 0.95 + luckBonus) quality = 'rare';
    else if (qualityRoll < 0.99 + luckBonus) quality = 'epic';
    else quality = 'legendary';
    
    // 按品质和玩家等级选择装备模板
    var available = EQUIP_BASES.filter(function(e) {
        return e.quality === quality && e.levelReq <= gameState.level;
    });
    if (available.length === 0) return null;
    var base = available[Math.floor(Math.random() * available.length)];
    
    // 生成实例，添加随机浮动
    return this.createEquipInstance(base);
};
```
