# 放置修仙 - 微信小游戏迭代策略

> 目标：从当前单文件 HTML Demo 迭代为可上线的微信小游戏
> 日期：2026-04-21

---

## 一、竞品调研结论

### 1.1 头部产品对标

| 产品 | 核心定位 | 关键成功要素 | 我们的差距 |
|------|---------|-------------|-----------|
| **寻道大千** | 砍树+放置修仙 | 极简操作、社交攀比、密集付费点 | 缺少社交/竞技系统、无付费设计 |
| **一念逍遥** | 水墨修仙挂机 | 国风美术、境界突破爽感、宗门社交 | 美术表现力不足、无多人互动 |
| **次神光之觉醒** | 放置+RPG | 爽感战斗、装备掉落、成长可视化 | 战斗系统过简、掉落反馈弱 |

### 1.2 放置修仙赛道共性规律

1. **核心循环**：砍怪/修炼 → 获得资源 → 提升境界 → 解锁新功能 → 继续砍怪
2. **留存三板斧**：七日任务、每日签到、限时活动
3. **付费阶梯**：便利性（月卡）→ 轻付费（抽奖）→ 重付费（累充/战令）
4. **社交驱动**：排行榜、宗门、PVP 是长线留存的核心

## 二、微信小游戏上线门槛

### 2.1 资质要求（必须提前准备）

| 资质 | 必要性 | 预计耗时 | 备注 |
|------|--------|---------|------|
| **软著（计算机软件著作权）** | 必须 | 1-3个月 | 提前申请，最耗时的环节 |
| **ICP备案** | 必须 | 10-30工作日 | 小游戏发布前必须完成 |
| **微信认证（企业主体）** | 必须 | 1-3工作日 | 个人主体无法开通支付 |
| **版号** | 如需内购则必须 | 6-12个月 | 仅广告变现可先不上 |
| **适龄提示** | 必须 | 1工作日 | 审核前置条件 |

**时间线估算**：
- 仅广告变现（IAA）：13-37工作日（不含软著申请时间）
- 含内购（IAP）：+ 版号时间（6-12个月）

### 2.2 技术限制

| 限制项 | 规则 | 当前状态 |
|--------|------|---------|
| 首包大小 | ≤ 4MB | ⚠️ 需评估（当前单文件+音乐可能超） |
| 总包大小 | ≤ 16MB | ✅ 目前远低于 |
| 内存限制 | iOS 约 200-300MB | ✅ 2D Canvas 轻量 |
| 渲染方案 | Canvas 2D / WebGL | ✅ 当前 DOM 需迁移 |

### 2.3 审核红线（必须规避）

- ❌ 强制分享、诱导分享朋友圈
- ❌ 自动播放音视频（需用户交互后）
- ❌ 无实质内容仅作为跳转跳板
- ❌ 血腥暴力、恐怖色情内容
- ❌ 赌博或宣扬赌博玩法
- ✅ 必须提供客服/投诉入口
- ✅ 必须接入中宣部实名认证系统

## 三、当前产品诊断

### 3.1 已具备的基础

| 模块 | 完成度 | 评价 |
|------|--------|------|
| 修炼系统 | 80% | 点击+自动闭关，核心循环已通 |
| 战斗系统 | 60% | 自动+手动 Boss，但爽感不足 |
| 装备系统 | 50% | 有掉落和强化，但深度不够 |
| 技能系统 | 60% | 6个技能，但策略性弱 |
| 境界系统 | 70% | 境界突破有爽感，但后续内容少 |
| 存档系统 | 80% | localStorage，需迁移到微信云 |
| BGM/音效 | 70% | 有背景音乐，但缺少战斗音效 |

### 3.2 核心短板

| 短板 | 影响 | 优先级 |
|------|------|--------|
| **无社交系统** | 无法形成传播和攀比，留存差 | P0 |
| **无付费设计** | 无法变现 | P0 |
| **美术表现力弱** | 首屏留存率低 | P1 |
| **内容深度不足** | 玩家很快到达内容边界 | P1 |
| **技术架构不匹配** | 当前 DOM 方案无法直接上线微信 | P0 |
| **缺少运营活动** | 无签到/七日/限时活动 | P1 |

## 四、迭代路线图

### Phase 1：技术迁移（2-3周）—— 必须先完成

**目标**：将当前 DOM 方案迁移为微信小游戏兼容架构

| 任务 | 具体工作 | 验收标准 |
|------|---------|---------|
| 框架选型 | 评估 Cocos Creator / Laya / 原生 Canvas | 确定技术方案文档 |
| 渲染迁移 | DOM → Canvas 2D 渲染 | 所有 UI 用 Canvas 绘制 |
| 存档迁移 | localStorage → 微信云存档 | 数据可跨设备同步 |
| 音频迁移 | HTML5 Audio → 微信音频 API | BGM/音效正常播放 |
| 包体优化 | 资源压缩、代码分割 | 首包 ≤ 4MB |
| 微信适配 | 登录、分享、广告 SDK 接入 | 基础微信能力可用 |

**技术建议**：
- 推荐 **Cocos Creator 3.x**，有成熟的微信小游戏发布流程
- 或保持 **原生 Canvas 2D**，用微信小游戏框架直接开发（学习成本低）

### Phase 2：核心玩法补强（2-3周）

**目标**：补齐放置修仙的核心系统，达到可测试水平

| 系统 | 设计要点 | 参考对标 |
|------|---------|---------|
| **洞府/挂机** | 离线收益可视化，洞府可升级提升收益倍率 | 一念逍遥洞府 |
| **灵兽系统** | 捕捉灵兽，灵兽可出战、升级、进化 | 寻道大千灵兽 |
| **炼丹系统** | 采集药材 → 炼丹 → 获得 buff/永久属性 | 传统修仙炼丹 |
| **装备重设计** | 增加套装效果、随机词条、品质可视化 | 次神装备系统 |
| **战斗爽感** | 伤害数字飘字、暴击特效、连击反馈 | 次神战斗 |
| **任务系统** | 主线任务引导 + 每日任务奖励 | 通用设计 |

### Phase 3：社交与竞技（2-3周）

**目标**：加入社交元素，提升留存和传播

| 系统 | 设计要点 | 微信能力 |
|------|---------|---------|
| **排行榜** | 战力榜、境界榜、财富榜 | 微信好友排行榜 |
| **宗门/帮派** | 创建/加入宗门，宗门任务、宗门战 | 微信群关联 |
| **PVP 斗法** | 异步 PVP（打玩家镜像），排名奖励 | 无需实时对战 |
| **世界聊天** | 全服聊天频道 | 内容安全审核 |
| **邀请奖励** | 邀请好友获得资源奖励 | 微信分享卡片 |

### Phase 4：商业化（2周）

**目标**：接入变现能力

| 变现方式 | 设计要点 | 技术准备 |
|---------|---------|---------|
| **广告变现（IAA）** | 看广告双倍收益、复活、加速 | 微信广告组件 |
| **月卡/特权** | 自动修炼、资源加成、去广告 | 虚拟支付（需版号） |
| **战令/通行证** | 免费+付费双轨奖励 | 虚拟支付（需版号） |
| **礼包/抽奖** | 限时礼包、灵兽抽奖 | 虚拟支付（需版号） |

**建议路径**：先上 IAA（广告变现），同步申请版号，版号下来后开 IAP（内购）。

### Phase 5：运营活动（持续）

| 活动类型 | 示例 | 目的 |
|---------|------|------|
| 新手期 | 七日登录、成长之路 | 建立习惯 |
| 每日 | 签到、每日任务 | 日活 |
| 周期 | 周末双倍、月末冲榜 | 周活 |
| 节日 | 春节/中秋限定活动 | 回流 |
| 限时 | 异兽入侵、法宝现世 | 付费刺激 |

## 五、关键决策点

### 决策 1：技术路线

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **Cocos Creator 3.x** | 成熟、跨平台、微信一键发布 | 学习成本、包体较大 | ⭐⭐⭐⭐ |
| **LayaAir** | 轻量、2D 性能好 | 社区较小 | ⭐⭐⭐ |
| **原生 Canvas 2D** | 完全可控、包体最小 | 需自研框架、开发慢 | ⭐⭐⭐ |
| **保持 DOM（不可行）** | - | 微信不支持 DOM | ❌ |

**建议**：如果团队有 Cocos 经验 → Cocos；如果追求快速上线 → 原生 Canvas 2D。

### 决策 2：美术风格

| 方案 | 优点 | 缺点 | 推荐度 |
|------|------|------|--------|
| **国风水墨** | 差异化、符合修仙主题 | 美术成本高 | ⭐⭐⭐⭐ |
| **Q版卡通** | 开发快、受众广 | 同质化严重 | ⭐⭐⭐ |
| **像素风** | 成本低、独特 | 小众 | ⭐⭐ |

**建议**：保持国风水墨方向，但用 AI 生成素材降低成本。

### 决策 3：上线节奏

| 阶段 | 内容 | 时间 |
|------|------|------|
| **MVP** | 核心循环 + 广告变现 | 2-3个月 |
| **正式版** | + 社交 + 付费 | 版号下来后 |
| **长线运营** | 持续内容更新 | 持续 |

## 八、总结

**当前状态**：有核心玩法雏形，但技术架构、社交系统、商业化均未就绪。

**关键路径**：
1. **立即申请软著**（最长耗时环节）
2. **技术迁移到 Canvas + 微信框架**（上线前提）
3. **先上 IAA MVP**（快速验证市场）
4. **版号下来后开 IAP**（正式商业化）

**预期时间**：
- IAA MVP 上线：2-3 个月（不含软著等待时间）
- IAP 完整版上线：软著 + 版号下来后（总计 6-12 个月）
